# 5.5 从 CC Switch 迁移

本页说明如何从 CC Switch 迁移到 `bianma-app` 文档口径与公开协议口径。

## 适用对象

- 曾使用 CC Switch 的个人或团队用户。
- 手里有历史文档、脚本、分享链接或自动化流程，仍引用旧命名或旧协议。

## 名称变化

- 新对外名称：`bianma-app`
- 历史名称：`CC Switch`（仅用于迁移和兼容说明）

在部分历史文件、历史路径或历史界面中，仍可能看到旧命名。

## 协议变化

- 当前公开主协议口径：`bianma://`
- 旧协议：`ccswitch://`（仅作为 legacy compatibility 讨论）

## 旧链接处理建议

1. 新文档和新示例统一使用 `bianma://`。
2. 历史链接保留在迁移期的兼容说明中，不再作为主入口宣传。
3. 对外分享的模板、教程、FAQ，优先替换为 `bianma://` 版本。

## 兼容状态说明

`ccswitch://` 的实际兼容状态以当前实现与版本行为为准。
如果你需要确认实际运行时兼容范围，请结合实现核对结果或版本说明，不建议仅凭历史文档判断。

## 推荐迁移动作清单

1. 更新团队内文档标题与仓库说明为 `bianma-app`。
2. 把对外示例链接替换为 `bianma://`。
3. 将 `ccswitch://` 相关内容收敛到迁移章节或历史兼容章节。
4. 在公共教程中补充“迁移说明链接”，避免用户误用旧协议。
