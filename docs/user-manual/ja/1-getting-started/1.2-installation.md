# 1.2 インストールガイド

このページでは、bianma-app の公開向けインストール入口とローカル開発入口を説明します。

公開ドキュメントの正本は [中文ユーザーマニュアル](../../zh/README.md) と [中文インストールガイド](../../zh/1-getting-started/1.2-installation.md) です。

## 前提条件

### 必須

- Node.js 18+
- pnpm 8+

### ローカル開発またはローカルビルド時

- Rust
- 各プラットフォーム向けの Tauri 依存関係

## Node.js のインストール

### Windows

1. [Node.js 公式サイト](https://nodejs.org/) から LTS 版をダウンロードします。
2. インストール後、次を実行します。

```bash
node --version
npm --version
```

### macOS

```bash
brew install node
```

### Linux（Ubuntu / Debian）

```bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

## 対応 CLI ツールのインストール

bianma-app の公開実装で現在扱う主なツール:

- Claude Code
- Codex CLI
- Gemini CLI
- OpenCode
- OpenClaw

代表的なインストール例:

```bash
npm install -g @anthropic-ai/claude-code
npm install -g @openai/codex
npm install -g @google/gemini-cli
npm i -g opencode@latest
npm i -g openclaw@latest
```

環境ごとの差異がある場合は、各ツールの上流公式手順も参照してください。

## プロジェクトをローカルで起動

リポジトリのルートで実行:

```bash
pnpm install
pnpm dev
```

ローカルで本番ビルドを行う場合:

```bash
pnpm build
```

## 公開リリースパッケージ

公開デスクトップ配布物は [Releases ページ](https://github.com/CreatorEdition/bianma-app/releases) にあります。

公開ブランドは `bianma-app` ですが、既存のビルドと配布チェーンの互換性のため、一部のデスクトップ成果物は `CC-Switch-*` という旧ファイル名を保持しています。

それらの名称は互換用識別子として扱い、主たる製品名としては扱わないでください。

## 初回確認

1. アプリが正常に起動する。
2. 「プロバイダー追加」画面に入れる。
3. 少なくとも 1 件のプロバイダー追加と有効化を完了できる。
