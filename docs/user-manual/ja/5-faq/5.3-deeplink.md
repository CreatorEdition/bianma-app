# 5.3 ディープリンクプロトコル（`bianma://`）

## 概要

`bianma://` は公開向けのディープリンクプロトコルです。

主な利用シーン:

- チーム内での設定共有
- チュートリアルでのワンクリック設定
- デバイス間の素早い同期

## プロトコル形式

### V1

```text
bianma://v1/import?resource={type}&app={app}&name={name}&...
```

共通パラメータ:

| パラメータ | 必須 | 説明 |
|---|---|---|
| `resource` | はい | `provider` / `mcp` / `prompt` / `skill` |
| `app` | いいえ | `claude` / `codex` / `gemini` / `opencode` / `openclaw` |
| `name` | はい | 表示名 |

### Provider パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `endpoint` | いいえ | API エンドポイント URL |
| `apiKey` | いいえ | API キー |
| `homepage` | いいえ | サイト URL |
| `model` | いいえ | デフォルトモデル |
| `config` | いいえ | Base64 エンコード済み設定 |
| `configFormat` | いいえ | `json` / `toml` |
| `configUrl` | いいえ | リモート設定 URL |
| `enabled` | いいえ | インポート後に有効化するか |

### Prompt パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `content` | はい | Base64 エンコード済みプロンプト本文 |
| `description` | いいえ | 説明 |
| `enabled` | いいえ | インポート後に有効化するか |

### MCP パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `apps` | はい | 対象アプリのカンマ区切り一覧 |
| `config` | はい | Base64 エンコード済み MCP JSON |
| `enabled` | いいえ | インポート後に有効化するか |

### Skill パラメータ

| パラメータ | 必須 | 説明 |
|---|---|---|
| `repo` | はい | `owner/name` 形式のリポジトリ |
| `directory` | いいえ | ディレクトリ |
| `branch` | いいえ | Git ブランチ |

## 例

Provider:

```text
bianma://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

MCP:

```text
bianma://v1/import?resource=mcp&apps=claude,codex&name=mcp-fetch&config=eyJtY3BTZXJ2ZXJzIjp7ImZldGNoIjp7ImNvbW1hbmQiOiJ1dngiLCJhcmdzIjpbIm1jcC1zZXJ2ZXItZmV0Y2giXX19fQ==
```

Prompt:

```text
bianma://v1/import?resource=prompt&app=claude&name=Code%20Review&content=IyBSb2xlCllvdSBhcmUgYSBzdHJpY3QgcmV2aWV3ZXIu
```

Skill:

```text
bianma://v1/import?resource=skill&name=my-skill&repo=owner/repo&directory=skills/my-skill&branch=main
```

## 生成方法

1. パラメータを準備します。
2. V1 形式で URL を組み立てます。
3. クエリ値は URL エンコードします。
4. `prompt` と `mcp` の本文は Base64 化してから埋め込みます。

```javascript
const params = new URLSearchParams({
  resource: "provider",
  app: "claude",
  name: "My Provider",
  endpoint: "https://api.example.com",
  apiKey: "sk-xxx",
});

const url = `bianma://v1/import?${params.toString()}`;
```

## 利用時の挙動

1. システムがアプリを開くか確認します。
2. 確認後にアプリが起動します。
3. インポート確認ダイアログが表示されます。
4. ユーザーが確認した後にのみインポートされます。

## プロトコル登録

インストール時に `bianma://` が登録され、旧スキームとの互換性も保持されます。

Windows では通常 `HKEY_CLASSES_ROOT\\bianma` を確認します。

## セキュリティ

- 実運用の API キーを含むリンクは公開しないでください。
- 信頼できる配布元のリンクだけを利用してください。
- アプリはリンク形式を検証し、インポート前にプレビューを表示します。

## Legacy Compatibility

`ccswitch://` は移行と後方互換のためだけに残されています。
新しい公開ドキュメントや外部連携では `bianma://` を使用してください。
