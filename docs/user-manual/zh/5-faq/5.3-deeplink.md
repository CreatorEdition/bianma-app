# 5.3 深度链接协议（`bianma://`）

本页定义 `bianma-app` 在公有仓文档中的公开深度链接协议口径。

## 协议用途

`bianma://` 用于通过链接导入公开可描述的配置资源，例如：

- `provider`
- `mcp`
- `prompt`
- `skill`

## 协议格式

```text
bianma://v1/import?resource={type}&app={app}&name={name}&...
```

## 通用参数

| 参数 | 必填 | 说明 |
|---|---|---|
| `resource` | 是 | 资源类型：`provider` / `mcp` / `prompt` / `skill` |
| `name` | 是 | 资源显示名称 |
| `app` | 否 | 目标应用：`claude` / `codex` / `gemini` / `opencode` / `openclaw` |

## 各资源常见参数

### `provider`

| 参数 | 必填 | 说明 |
|---|---|---|
| `endpoint` | 否 | API 端点地址 |
| `apiKey` | 否 | API Key |
| `model` | 否 | 默认模型 |
| `notes` | 否 | 备注 |

### `mcp`

| 参数 | 必填 | 说明 |
|---|---|---|
| `apps` | 否 | 逗号分隔的应用列表 |
| `config` | 是 | MCP 配置（Base64 编码 JSON） |

### `prompt`

| 参数 | 必填 | 说明 |
|---|---|---|
| `content` | 是 | Base64 编码后的提示词正文 |
| `description` | 否 | 描述 |

### `skill`

| 参数 | 必填 | 说明 |
|---|---|---|
| `repo` | 是 | 仓库，格式 `owner/name` |
| `directory` | 否 | 子目录 |
| `branch` | 否 | 分支名 |

## 示例

### 导入 Provider

```text
bianma://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

### 导入 MCP

```text
bianma://v1/import?resource=mcp&name=mcp-fetch&apps=claude,codex&config=eyJtY3BTZXJ2ZXJzIjp7ImZldGNoIjp7ImNvbW1hbmQiOiJ1dngiLCJhcmdzIjpbIm1jcC1zZXJ2ZXItZmV0Y2giXX19fQ==
```

### 导入 Prompt

```text
bianma://v1/import?resource=prompt&app=claude&name=Code%20Review&content=IyBSb2xlCllvdSBhcmUgYSBzdHJpY3QgcmV2aWV3ZXIu
```

### 导入 Skill

```text
bianma://v1/import?resource=skill&name=my-skill&repo=owner/repo&directory=skills/my-skill&branch=main
```

## 安全建议

- 不要公开分享包含真实 `apiKey` 的链接。
- 导入前先校验链接来源与参数含义。
- 建议在团队内使用最小权限的测试密钥。

## 工具支持状态（公有仓当前实现）

已支持：

- Claude Code（`app=claude`）
- Codex CLI（`app=codex`）
- Gemini CLI（`app=gemini`）
- OpenCode（`app=opencode`）
- OpenClaw（`app=openclaw`）

部分支持或待澄清：

- 无（针对当前文档已列出的公开 `app` 值）。
- 若提到“本轮关注范围”，仅表示讨论对象，不代表其余已列出 `app` 不支持。

未支持（公有仓当前实现）：

- Cursor
- Windsurf
- Cline

未列入“已支持”的工具，不应按已公开支持理解。
