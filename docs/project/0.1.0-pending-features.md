# 0.1.0 待处理功能

本文档用于记录已经确认方向、但不在当前轮次实现范围内的公开仓功能项。

## 桌面原生材质化壳层

- 状态：待处理
- 目标版本：`0.1.0`
- 范围：公开仓桌面端视觉升级

### 功能目标

在不引入私有产品层能力的前提下，为桌面端补齐更现代的原生材质化窗口壳层：

- Windows：优先采用 `Mica Alt`
- macOS：优先采用 `NSVisualEffectView`

### 预期效果

- 让窗口标题区、侧边栏、顶部工具区更接近现代原生桌面应用观感
- 保持 `bianma-app` 的公开仓界面风格不再停留在旧式纯平冷色壳层
- 跟随系统主题、壁纸和系统材质变化，获得自然的轻微色彩联动

### 第一阶段建议范围

- Windows：
  - 主窗口背景壳层使用 `Mica Alt`
  - 避免把 `Acrylic` 作为整窗主背景
- macOS：
  - 侧边栏、标题区、工具区采用 `NSVisualEffectView`
  - 让材质表现跟随系统深浅色和窗口激活状态
- Web 内容层：
  - 只做轻量半透明与弱模糊叠层
  - 主内容区保持高可读性，不做整页重毛玻璃

### 非目标

- 不在当前轮次实现
- 不引入私有发布、私有 updater、私有主题策略
- 不做定时自动换色效果
- 不把 `ccswitch://` 或旧品牌视觉重新抬回主口径
- 不要求 Linux 首阶段同步具备同等级原生材质效果

### 实现边界

该功能后续应交由公有仓编码 Agent 处理，当前文档 Agent 只记录规划边界。

- 可做：
  - Tauri 桌面窗口材质能力接入
  - Web 层中性半透明 token 调整
  - 不支持平台的安全降级
- 不在本轮处理：
  - 具体平台实现代码
  - 大规模视觉重构
  - 私有产品差异化主题

### 验收原则

- 不支持原生材质的平台必须自动降级为普通实底背景
- 配置页、列表页、表单页、代码区域可读性优先
- 材质层只用于“窗口壳层”，不用于整个主内容区
- Windows 与 macOS 表现允许不同，但视觉方向应一致
