# 1.2 Installation Guide

This page covers the public installation and local development entry for bianma-app.

If you need the canonical public entry, start with the [Chinese user manual](../../zh/README.md) and the [Chinese installation guide](../../zh/1-getting-started/1.2-installation.md).

## Prerequisites

### Required

- Node.js 18+
- pnpm 8+

### For local development or local builds

- Rust
- Platform-specific Tauri dependencies

## Install Node.js

### Windows

1. Download the LTS version from the [Node.js website](https://nodejs.org/).
2. After installation, run:

```bash
node --version
npm --version
```

### macOS

```bash
brew install node
```

### Linux (Ubuntu / Debian)

```bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

## Install Supported CLI Tools

bianma-app currently manages public integrations for:

- Claude Code
- Codex CLI
- Gemini CLI
- OpenCode
- OpenClaw

Typical installation examples:

```bash
npm install -g @anthropic-ai/claude-code
npm install -g @openai/codex
npm install -g @google/gemini-cli
npm i -g opencode@latest
npm i -g openclaw@latest
```

Refer to each upstream tool for its own installation details if your environment differs.

## Run the Project Locally

From the repository root:

```bash
pnpm install
pnpm dev
```

If you need a local production build:

```bash
pnpm build
```

## Public Release Packages

Public desktop release assets are published on the [Releases page](https://github.com/CreatorEdition/bianma-app/releases).

The public brand is `bianma-app`, but some desktop artifacts still keep legacy `CC-Switch-*` file names for compatibility with the existing build and packaging chain.

Treat those file names as compatibility identifiers rather than the primary product name.

## First Validation

1. The app launches successfully.
2. You can enter the Add Provider view.
3. You can complete at least one provider add + enable flow.
