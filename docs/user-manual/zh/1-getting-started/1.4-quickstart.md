# 1.4 快速上手

目标：在 5 分钟内跑通第一条可用链路。

## 第一步：添加第一个供应商

1. 点击主界面右上角的 **+** 按钮，或使用“添加供应商”入口。
2. 在“预设”下拉框中选择目标应用或供应商。
   - 常见目标应用：Claude Code、Codex、Gemini CLI、OpenCode、OpenClaw
   - 常用预设：智谱 GLM、MiniMax、DeepSeek、Kimi、PackyCode
   - 如无合适预设，可选择“自定义”手动配置
3. 填写基础配置，例如官方登录或 API Key。
4. 点击“添加”并返回供应商列表。

![image-20260108002807657](../../assets/image-20260108002807657.png)

> 💡 预设会自动填充端点地址，你通常只需要补充 API Key 或完成官方登录。

## 第二步：启用供应商

添加完成后，供应商会出现在列表中。

**方式一：主界面切换**
- 点击供应商卡片的“启用”按钮

**方式二：托盘快速切换**
- 右键系统托盘图标
- 直接点击供应商名称

如使用 CLI 终端，建议切换后重开当前终端会话，以确保配置生效。

## 第三步：做最小验证

切换供应商后，各 CLI 工具的生效方式不同：

| 应用 | 生效方式 |
|------|----------|
| Claude Code | ✅ 即时生效（支持热重载） |
| Codex | 需要关闭并重新打开终端 |
| Gemini | ✅ 即时生效（每次请求重新读取配置） |
| OpenCode | 需要关闭并重新打开终端 |
| OpenClaw | 需要关闭并重新打开终端 |

### Claude Code 首次安装提示

如果 Claude Code 首次启动时提示需要登录或显示初始化引导，可在 `bianma-app` 中开启“跳过 Claude Code 初次安装确认”选项：

1. 打开 `bianma-app`“设置 → 通用”
2. 开启“跳过 Claude Code 初次安装确认”开关
3. 重新启动 Claude Code

![image-20260108002626389](../../assets/image-20260108002626389.png)

> ⚠️ 此选项会写入 `~/.claude/settings.json` 的 `skipIntroduction` 字段，以跳过官方的新手引导流程。

## 验证配置

启动任一已配置的 CLI 工具，并输入一个简单测试问题：

```bash
# Claude Code
claude
> 你好，请简单介绍一下自己

# Codex
codex
> 你好，请简单介绍一下自己

# Gemini
gemini
> 你好，请简单介绍一下自己

# OpenCode
opencode
> 你好，请简单介绍一下自己

# OpenClaw
openclaw
> 你好，请简单介绍一下自己
```

如果 AI 能正常回复，说明基础配置已经可用。

## 常见问题

### 切换后不生效？

请确认已经重启终端或对应 CLI。配置文件在切换时已更新，但运行中的程序不会自动重新加载。

### 找不到预设？

如果你的供应商不在预设列表中，选择“自定义”手动配置。可参考 [添加供应商](../2-providers/2.1-add.md)。

### 如何恢复官方登录？

选择“官方登录”预设（Claude / Codex）或“Google 官方”预设（Gemini），重启客户端后按登录流程操作。
