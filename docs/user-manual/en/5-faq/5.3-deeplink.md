# 5.3 Deep Link Protocol (`bianma://`)

## Overview

`bianma://` is the public deep link protocol for one-click imports.
Legacy compatibility notes appear in the Legacy Compatibility section below.

Use cases:

- Team configuration sharing
- One-click setup in tutorials
- Quick sync across devices

## Protocol Format

### V1

```text
bianma://v1/import?resource={type}&app={app}&name={name}&...
```

Common parameters:

| Parameter | Required | Description |
|---|---|---|
| `resource` | Yes | `provider` / `mcp` / `prompt` / `skill` |
| `app` | No | `claude` / `codex` / `gemini` / `opencode` / `openclaw` |
| `name` | Yes | Display name |

### Provider parameters

| Parameter | Required | Description |
|---|---|---|
| `endpoint` | No | API endpoint URL, supports comma-separated multiple URLs |
| `apiKey` | No | API key |
| `homepage` | No | Provider website |
| `model` | No | Default model |
| `haikuModel` | No | Claude Haiku model |
| `sonnetModel` | No | Claude Sonnet model |
| `opusModel` | No | Claude Opus model |
| `notes` | No | Notes |
| `icon` | No | Built-in icon name |
| `config` | No | Base64-encoded configuration content |
| `configFormat` | No | `json` / `toml` |
| `configUrl` | No | Remote configuration URL |
| `enabled` | No | Whether to enable after import |

### Prompt parameters

| Parameter | Required | Description |
|---|---|---|
| `content` | Yes | Base64-encoded prompt content |
| `description` | No | Description |
| `enabled` | No | Whether to enable after import |

### MCP parameters

| Parameter | Required | Description |
|---|---|---|
| `apps` | Yes | App list, comma-separated |
| `config` | Yes | Base64-encoded MCP JSON |
| `enabled` | No | Whether to enable after import |

### Skill parameters

| Parameter | Required | Description |
|---|---|---|
| `repo` | Yes | Repository in `owner/name` format |
| `directory` | No | Directory path |
| `branch` | No | Git branch |

## Import Examples

Provider:

```text
bianma://v1/import?resource=provider&app=claude&name=My%20Provider&endpoint=https%3A%2F%2Fapi.example.com&apiKey=sk-xxx
```

MCP:

```text
bianma://v1/import?resource=mcp&apps=claude,codex&name=mcp-fetch&config=eyJtY3BTZXJ2ZXJzIjp7ImZldGNoIjp7ImNvbW1hbmQiOiJ1dngiLCJhcmdzIjpbIm1jcC1zZXJ2ZXItZmV0Y2giXX19fQ==
```

Prompt:

```text
bianma://v1/import?resource=prompt&app=claude&name=Code%20Review&content=IyBSb2xlCllvdSBhcmUgYSBzdHJpY3QgcmV2aWV3ZXIu
```

Skill:

```text
bianma://v1/import?resource=skill&name=my-skill&repo=owner/repo&directory=skills/my-skill&branch=main
```

## Generating Links

1. Prepare the parameters.
2. Follow the V1 format.
3. URL-encode query values.
4. Base64-encode prompt and MCP payloads before embedding them in the URL.

```javascript
const params = new URLSearchParams({
  resource: "provider",
  app: "claude",
  name: "My Provider",
  endpoint: "https://api.example.com",
  apiKey: "sk-xxx",
});

const url = `bianma://v1/import?${params.toString()}`;
```

## Import Flow

Clicking a supported deep link will:

1. Ask the system to open the app.
2. Open the app after confirmation.
3. Show an import confirmation dialog.
4. Let the user confirm or cancel before import.

## Protocol Registration

The app registers `bianma://` during installation.

If protocol registration is broken:

**macOS**

```bash
/usr/bin/open -a "bianma-app" --args --register-protocol
```

**Windows**

```text
HKEY_CLASSES_ROOT\bianma
```

**Linux**

Check the `MimeType` configuration in the `.desktop` file.

## Security Notes

- Do not share links containing real production API keys.
- Only import configurations from trusted sources.
- The app validates the link format and shows a preview before import.

## Legacy Compatibility

`ccswitch://` is kept only for migration and backward compatibility.
New public integrations and documentation should use `bianma://`.
Refer to the [migration compatibility guide (ZH)](../../zh/5-faq/5.5-migration-compatibility.md) for the current compatibility scope.
